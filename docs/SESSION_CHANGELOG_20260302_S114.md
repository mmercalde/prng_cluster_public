### 4. Patch 3 — Optuna Study Resume Flag (--resume-study)
**Files modified:**
- `window_optimizer.py` — added `--resume-study` CLI argument
- `window_optimizer_bayesian.py` — added `resume_study: bool = False` to `search()`
- `agent_manifests/window_optimizer.json` — documented param, fixed trials 50→100

**Default behavior: UNCHANGED** — fresh study created every run (safe, clean audit artifacts)

**With flag:** resumes most recent incomplete study DB, continues from checkpoint

**Usage:**
```bash
# Fresh run (default — no flag needed)
--params '{"lottery_file": "daily3.json"}'

# Resume most recent incomplete study
--params '{"lottery_file": "daily3.json", "resume_study": true}'
```

**Resume logic:**
1. Scans `optuna_studies/window_opt_*.db` sorted by mtime
2. Loads most recent DB
3. Checks: completed > 0 AND completed < max_iterations
4. If resumable: sets `load_if_exists=True`, skips warm-start enqueue
5. Calculates `_trials_to_run = max_iterations - completed`
6. If not resumable: falls back to fresh study with warm-start

**Why flag not auto-resume:**
- Auto-resume risks loading stale studies from weeks ago
- PRNG type changes, data changes, threshold changes would corrupt TPE model
- Maintains "each run is a clean audit artifact" principle
- Consistent with existing WATCHER params architecture

**Patcher script:** `apply_resume_patch_v2.py`
